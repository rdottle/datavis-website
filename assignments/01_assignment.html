Hive Plots

What is a hive plot?

Hive plots are one representational and computational solution to visualizing complex networks. Networks generally consist of nodes, which are connected within a system by edges. Hive plots place nodes on "radially oriented" linear axes, like a coordinate system. Unlike other network visualization techniques, hive plots project the network onto a multi-dimensional coordinate system that is based on properties of the network, like clustering or connectivity. This allows plot readers to spot patterns in the given network.

Why a hive plot?

Network visualizations are difficult to interpret, and often if the network is large, visualizations begin to resemble hairballs. While visually appealing, network representations can be difficult to interpet. Visualizing networks is important, though and networks are utilized in a variety of fields to model dynamic relationships between units, or nodes. They enable the representation of systems, and those systems’ global properties. Network analysis can occur at the genome, cell, social circle, or city scale. Regardless of the size of the network, communicating its structure and modeling network behaviours can be quite difficult.

Enter hive plots. Hive plots allow readers to understand aspects of a network’s structure. Unlike hairballs, hive plots manage visualizing large numbers of edges and nodes, structured based on network parameters of interest like connectivity.

Hive plots were developed to address issues of visualizing large networks, when a hairball visualization does not adequately present patterns. Hive plots offer the opportunity to visualize network structures in a way hairballs do not. They also allow users to tune the representation to represent a specific parameter of interest. Hive plots follow axes, and the patterns presented are due to underlying data, and not algorithmic interpretation, which is an advantage of using the hive plot over other visualization methods.

How are hive plots created?

To make a hive plot, network features must first be calculated. These network features may be based in network graph theory, and could be measures of clustering, or global or local connectivity. These parameters are chosen based on the hive plot creator’s network pattern interests. Once a network parameter is chosen, the node coordinates used in the hive plot are then based on that parameter. Unlike in a graph network representation, nodes are represented as “glyphs” and edges as “curves” between nodes. Hive plots were first designed in 2010, as a more rational, quantitative and visually informative method of network visualization, specific to the fields of genomics. They have been used for many different forms of network analysis, and are appropriate for visualizing network parameters like centrality (betweenness, closeness), density, or clustering. Furthermore, the use of an axis (usually three) provides organization in a way that other network visualizations do not.

Hive Plots can be created in various software with network data in the format of nodes and edges. R offers a hive plot package,Python has a library, and hive plots can also be creared in d3.js and jhive for Java.
